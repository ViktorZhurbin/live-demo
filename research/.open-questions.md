# Open questions

Date: 2026-07-23. Things investigated far enough to have an answer worth keeping, but
deliberately not acted on. Each entry records what was measured, why it's parked, and
what would have to change to unpark it — so none of it gets re-derived from scratch.

Decisions that _were_ acted on live in `decisions/`. Ideas that were rejected live in `research/.closed-questions.md`.

## Method notes worth reusing

- **Measure brotli against the real CDN**, not gzip and not offline quality 11. The
  `@babel/core` swap looked like ~90 KB gzip and was near-zero brotli. Raw bytes are the
  only quality-independent number for cross-method comparison.
- Local `pnpm preview` does not compress `.wasm`; Cloudflare does. An early measurement
  overstated the compiler bucket by ~320 KB because of this.
- **A green `check:all` is not sufficient** for anything touching the compiler
  pipeline. Some issues only surface by executing real demos in a browser.

## The frame: this project's scale

Nearly zero users (see the root `CLAUDE.md`). Most items below were parked because
their justification was payload for readers who don't exist. That makes "does this
make the code simpler?" the deciding question almost every time, and it's why the
Rollup removal shipped while everything here did not. Re-read that framing before
reviving anything on payload grounds.

## Transpiler revisit

Full detail in `research/transpiler-research.md`. Watch for any of:

- **Yuku** shipping a JSX-automatic-runtime transform and an ESM→CJS transform. It
  already has the smallest footprint, the only wasm delivery story here that doesn't
  need cross-origin-isolation headers, and free structured import specifiers. Only the
  transform layer is missing.
- **oxc** publishing a non-threaded browser build of the transformer plus a CJS output
  mode. Best error reporting measured, and the toolchain this repo already trusts at
  build time.
- **Sucrase** returning to a real release cadence. 10x smaller than Babel, but no AST
  access (needs a second parse) and no codeframe.
- **Pre-bundling Babel into the package's own `dist/`** with node-builtin shims applied
  at package build time. Never tested. It's the one path that would make the stashed
  `@babel/core` work adoptable, since it moves the polyfill burden off consumers.
- **`@rolldown/browser` was never evaluated.** The original architecture review
  suggested it as a single-tool replacement for Babel + Rollup. The transpiler research
  measured `oxc-transform`, which is a different package with a different browser build,
  so its COOP/COEP disqualification doesn't automatically transfer. The bundling half of
  the pitch is moot now (`decisions/0001` removed bundling from the runtime), but its
  oxc-based transform in a browser wasm build is still an untested candidate. Check its
  browser build for shared `WebAssembly.Memory` first — that's the fast disqualifier —
  then whether it can emit CJS.
